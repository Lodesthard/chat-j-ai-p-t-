0. MODE D'EMPLOI RAPIDE POUR LANCER LE PROJET
================================================================================

1. Verifier les prerequis :

   - Java installe ;
   - Maven installe ;
   - un token de bot Telegram ;
   - une cle OpenWeatherMap.

2. Configurer le fichier :

   src/main/resources/application.properties

   Exemple de configuration :

   server.port=9090
   api.access.token=interop-secret
   telegram.api.url=https://api.telegram.org/bot
   telegram.bot.id=<TOKEN_TELEGRAM_BOT>
   open.weather.api.url=https://api.openweathermap.org/data/2.5/weather
   open.weather.forecast.url=https://api.openweathermap.org/data/2.5/forecast
   open.weather.api.token=<CLE_OPENWEATHERMAP>

3. Ouvrir un terminal a la racine du projet :

   c:\Users\bidbi\Documents\Interoperabilite\chat-j-ai-p-t-

4. Compiler le projet :

   mvn clean compile

   Cette commande genere aussi les classes Java depuis l'OAS :

   src/main/resources/openapi/chatbot-api.yaml

5. Lancer l'application :

   mvn spring-boot:run

6. Si le port 9090 est deja utilise, lancer sur un autre port :

   mvn spring-boot:run '-Dspring-boot.run.arguments=--server.port=9091'

7. Tester rapidement l'API REST :

   PowerShell :

   $headers = @{ 'X-API-Token' = 'interop-secret' }
   Invoke-WebRequest -UseBasicParsing -Headers $headers -Uri http://localhost:9090/joke

   Si vous avez lance sur 9091, remplacer 9090 par 9091.

   Dans un navigateur, utiliser le token directement dans l'URL :

   http://localhost:9090/joke?access_token=interop-secret

8. Tester le chatbot Telegram :

   - demarrer l'application ;
   - ouvrir Telegram ;
   - envoyer un message au bot, par exemple :

     meteo Paris

   Le listener Telegram interroge getUpdates toutes les 5 secondes. La reponse
   peut donc prendre quelques secondes.

9. Envoyer un message Telegram depuis l'API :

   D'abord, envoyer un message au bot depuis Telegram, puis recuperer le chat_id :

   Invoke-WebRequest -UseBasicParsing -Headers $headers -Uri http://localhost:9090/chats

   Ensuite envoyer un message :

   $body = @{
     chat_id = 123456789
     text = "Message envoye depuis l'API REST"
   } | ConvertTo-Json

   Invoke-WebRequest `
     -UseBasicParsing `
     -Headers $headers `
     -Method POST `
     -ContentType "application/json" `
     -Body $body `
     -Uri http://localhost:9090/message

10. Arreter l'application :

   Dans le terminal ou l'application tourne :

   Ctrl + C


1. PRESENTATION
================================================================================

Ce projet est une application Spring Boot qui expose les fonctionnalites d'un
chatbot Telegram de deux manieres :

- par Telegram, avec une boucle de polling basee sur getUpdates ;
- par une API REST securisee par token, decrite par un contrat OpenAPI (OAS).

Les deux entrees utilisent la meme logique metier :

- meteo actuelle d'une ville ;
- previsions meteo ;
- blague aleatoire, mieux notee ou moins bien notee ;
- consultation, ajout, modification et suppression de blagues ;
- envoi de messages Telegram depuis l'API REST.

Une blague possede, comme demande dans le sujet : un id, un titre, un texte et
une note sur 10.

Technologies principales :

- Java 8 configure dans Maven ;
- Spring Boot 2.4.5 ;
- Maven ;
- Telegram Bot API ;
- OpenWeatherMap API ;
- OpenAPI Generator Maven Plugin ;
- Lombok pour les modeles Telegram existants.

Port par defaut :

  http://localhost:9090

--------------------------------------------------------------------------------

POST /message

    "chat_id": 123456789,
    "text": "Bonjour depuis l'API REST"

  {
    "ok": true,
    "message_id": 42,
    "chat_id": 123456789,
    "text": "Bonjour depuis l'API REST"
  }

  HTTP 400
  {
    "status": 400,
    "error": "Bad Request",
    "message": "Erreur Telegram : ..."

7. CHATBOT TELEGRAM
================================================================================

Au demarrage de l'application, ListenerUpdateTelegram lance un Timer qui execute
un polling toutes les 5 secondes.

Flux :

  1. Appel a Telegram :

       GET https://api.telegram.org/bot<TOKEN>/getUpdates?offset=<offset>

  2. Pour chaque Update recu :

       - seuls les evenements de type message sont traites ;
       - les messages sans texte sont ignores ;
       - le texte est transmis a ChatbotService ;
       - la reponse est envoyee avec TelegramService.

  3. Apres traitement :

       offset = update_id + 1

Ce offset evite de retraiter les memes evenements au polling suivant.

Limite :

  L'offset est conserve en memoire. Si l'application redemarre, il repart a 0.
  Telegram peut donc renvoyer des evenements recents deja traites. Pour une
  version plus robuste, il faudrait persister le dernier offset dans un fichier
  ou une base de donnees.

Reponse ciblee :

  Le listener envoie les reponses avec reply_to_message_id egal au message_id du
  message entrant. Telegram affiche donc la reponse comme une reponse citee au
  message de l'utilisateur.


8. COMMANDES TELEGRAM
================================================================================

Meteo :

  meteo
  meteo Paris
  meteo Lyon

Previsions :

  previsions
  previsions Bordeaux

Blagues (store interne) :

  blague           (blague au hasard)
  blague top       (la mieux notee ; aussi : super, bonne)
  blague nulle     (la moins bien notee ; aussi : pourrie)
  blague 3         (blague d'id 3)

Blague Carambar (API externe) :

  joke             (blague tiree de l'API carambar-jokes)

  Cette commande appelle GET <carambar.api.url>/random. Par defaut l'instance
  hebergee est utilisee :

    https://carambar-jokes-79ge.onrender.com/api/random

  Aucun service local a lancer. Pour utiliser l'instance locale (projet Node
  carambar-jokes lance via npm start), changer carambar.api.url dans
  application.properties vers http://localhost:3000/api.

  La commande "blague" reste independante et utilise le store interne.

Ajouter une blague :

  ajouter blague Titre | Texte | Note(0-10)

  La note est optionnelle (defaut 5). Exemple :

  ajouter blague Le cafe | Pourquoi les devs aiment le cafe ? Java. | 6

Modifier une blague :

  modifier blague 3 Nouveau titre | Nouveau texte | 9

Supprimer une blague :

  supprimer blague 3

Tout autre message renvoie l'aide :

  Commandes disponibles :
  - meteo <ville>
  - previsions <ville>
  - blague
  - blague top (la mieux notee)
  - blague nulle (la moins bien notee)
  - blague <id>
  - joke (blague de l'API locale Carambar)
  - ajouter blague Titre | Texte | Note(0-10)
  - modifier blague <id> Titre | Texte | Note(0-10)
  - supprimer blague <id>


9. GESTION DES ERREURS
================================================================================

Les erreurs REST sont centralisees par GlobalExceptionHandler.

Format standard :

  {
    "status": 404,
    "error": "Not Found",
    "message": "..."
  }

Cas geres :

- token absent ou invalide : HTTP 401 ;
- blague inexistante : HTTP 404 ;
- payload invalide ou champ obligatoire manquant : HTTP 400 ;
- chat Telegram inexistant : HTTP 400 ou HTTP 502 selon la reponse Telegram ;
- reponse Telegram illisible : HTTP 502 ;
- erreur non prevue : HTTP 500.

Cote Telegram, les erreurs d'envoi sont loggees et ne font pas planter le
polling.

11.1 Configuration du token
--------------------------------------------------------------------------------

Toutes les requetes necessitent le token. Deux options :

Option A — en-tete (recommandee) :
  Dans l'onglet Headers de chaque requete :

    Key   : X-API-Token
    Value : interop-secret

Option B — variable d'environnement Postman (plus pratique) :

  1. Cliquer sur "New Environment" > nommer "Chatbot Local".
  2. Ajouter une variable :

       Variable : token
       Value    : interop-secret

  3. Dans chaque requete, onglet Headers :

       Key   : X-API-Token
       Value : {{token}}

  4. Selectionner l'environnement "Chatbot Local" en haut a droite.

14. LIMITES CONNUES
================================================================================

- Les blagues sont stockees en memoire : elles sont perdues au redemarrage.
- L'offset Telegram est stocke en memoire : un redemarrage peut retraiter des
  messages recents.
- Le polling se fait toutes les 5 secondes : une reponse Telegram peut donc
  avoir jusqu'a 5 secondes de delai.
- Le projet utilise un token REST simple, suffisant pour l'exercice mais pas
  adapte a un environnement de production.
- Le token Telegram et la cle OpenWeatherMap sont dans application.properties ;
  en production, ils devraient etre fournis par variables d'environnement.
